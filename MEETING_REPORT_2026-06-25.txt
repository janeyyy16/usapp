A. TICKET LIST + SERVICEPOWER SYNC

  1. Hourly auto-sync from ServicePower
     - Pulls this-month → today, scoped to San Antonio + Asheville covered
       zips. Skips tickets already in our DB so we never duplicate.
     - Throttled to once-per-hour per browser; runs silently on Ticket
       List mount. Removed the manual "Sync now" card.
     - Initial AHS status auto-set from SP call status:
         Cancelled  -> CL-Cancelled
         Completed  -> CL-Ready to Complete
         Accepted+NSA -> CSR-Needs Scheduling
         Accepted   -> CSR-Assigned to ASC
       Existing local status is preserved on re-sync.

  2. Hardened against duplicate tickets
     - Added DB unique index on (company_id, ticket_no).
     - In-process per-ticket lock; race-recovery on Postgres 23505
       to avoid duplicate inserts when two tabs sync at once.

  3. Branch / Loc resolver fixes
     - Resolver now goes through lookupZip(), which respects the
       Location Management overrides loaded from Supabase. Earlier
       it only saw the static autogenerated zip table.
     - Added city-level override (e.g. Salem -> Asheville) for
       cities the autogenerated table mis-routes.
     - Added a one-time per-session backfill: any ticket whose Loc
       is blank, "Unknown", or a stale SP service-location code
       (IH/OS/DO) gets re-resolved from the customer's zip + city +
       state and written back.
     - Sanitizer at read time strips garbage Loc values so the UI
       no longer shows "IH" / "Unknown".

  4. Work Order Source mapping
     - Mapped raw ServicePower MfgId codes to readable labels
       (e.g. K100 -> SPPN, I698 -> AIG WARRANTY, I250 -> ALLIANCE
       SPEED QUEEN, I990 -> ASSURANT SOLUTIONS, etc.).
     - Read-time remap: existing rows with raw codes display the
       friendly label immediately, no re-sync needed.
     - Moved the map to its own leaf module to break a circular
       import that was causing 500s on the dev server.

  5. Per-column filters on Ticket List
     - Added a small funnel icon next to every header. Clicking it
       opens a checkbox dropdown of every distinct value present in
       the filtered table, plus a search box. Excel-style autofilter.
     - Filters combine with existing Repair Status / Location /
       Source / Date / Search inputs (case-insensitive).
     - Date filter now accepts ISO (YYYY-MM-DD), MM/DD/YY, and
       MM/DD/YYYY (was only MM/DD/YY).
